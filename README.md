\## Что такое Git



Git — это распределённая система контроля версий.  

Она позволяет:

\- сохранять историю изменений файлов;

\- возвращаться к предыдущим версиям;

\- работать с проектом локально и синхронизироваться с удалённым репозиторием.

\## Базовый рабочий цикл

```text

Изменил файлы → git add → git commit → git push
```

\## Основные команды

\### Инициализация репозитория

git init

\### Проверка состояния

git status

\### Добавление файлов в staging

git add файл

git add .

\### Создание коммита

git commit -m "Сообщение коммита"

\### Просмотр истории

git log

\### Добавление удалённого репозитория

git remote add origin git@github.com:username/repository.git

\### Отправка изменений на GitHub

git push -u origin main

\### Клонирование репозитория

git clone git@github.com:username/repository.git

## Базовая последовательность работы с Git

Ниже описан стандартный порядок действий при работе с Git и GitHub.

Сначала в папке проекта создаётся локальный репозиторий командой `git init`.  

После этого можно проверить состояние файлов командой `git status`.

Чтобы подготовить файлы к сохранению, их добавляют в staging:  

`git add файл` — для одного файла,  

`git add .` — для всех файлов сразу.

После добавления файлов создаётся коммит, который фиксирует текущее состояние проекта в истории Git:  

`git commit -m "Описание изменений"`.

Для синхронизации с GitHub добавляется удалённый репозиторий:  

`git remote add origin git@github.com:username/repository.git`.

Первая отправка изменений на GitHub выполняется командой:  

`git push -u origin master`  

(или `main`, если основная ветка называется `main`).  

Эта команда одновременно отправляет коммиты и связывает локальную ветку с удалённой.

После первичной настройки для дальнейшей работы достаточно использовать команды:  

`git push` — для отправки изменений на GitHub,  

`git pull` — для получения изменений из удалённого репозитория.

Краткая памятка:  

Изменил файлы → git add → git commit → git push
